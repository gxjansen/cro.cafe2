Language Dropdown Test Instructions

Testing Procedure

  1. Open the built site at http://localhost:3000 or npm run preview
  2. Navigate to any page (e.g., /en/episodes/)
  3. Click on an episode to play it (audio player should appear at bottom)
  4. While audio is playing, navigate to another page using the navigation links
  5. Try to click the language dropdown button in the header

Expected Behavior

✅ Language dropdown button should be clickable on all pages
✅ Dropdown menu should appear when button is clicked
✅ Language options should be clickable and navigate correctly
✅ Dropdown should work even when audio player is visible
✅ Dropdown should work after navigating between pages

Fixes Applied

  • View Transition Support: Added transition:name="language-switcher" and transition:animate="none" to keep dropdown in transition layer
  • Event Delegation: Using document-level event listeners with capture phase for reliability
  • Z-Index Management: Dynamic z-index boosting when dropdown is active (z-index: 9999)
  • Isolation Context: Added isolation: isolate to create proper stacking context
  • Touch Support: Added touchstart event handling for mobile devices
  • Post-Navigation Fix: Automatic z-index correction after View Transitions complete

Browser Console Commands

Use these commands in the browser console to debug:

// Check if language switcher manager is initialized
console.log(window.__languageSwitcherManager);

// Check z-index of dropdown elements
document.querySelectorAll('.language-switcher-container').forEach(el => {
    console.log('Container z-index:', getComputedStyle(el).zIndex);
});

// Check if event delegation is working
document.querySelector('[data-language-button]').click();

// Force fix z-index issues
window.__languageSwitcherManager?.fixZIndexAfterTransition();

Common Issues and Solutions

Issue: Dropdown not clickable after navigation
Solution: View Transitions create new stacking contexts. We've added transition attributes to include the dropdown in the transition layer.
Issue: Click events not firing
Solution: Using event delegation with capture phase and stopImmediatePropagation to ensure events are caught.
Issue: Z-index conflicts with audio player
Solution: Dynamic z-index boosting when dropdown is active (9998/9999) ensures it stays above everything.
